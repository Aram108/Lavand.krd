Recon & Enumeration

Subdomain discovery and directory fuzzing commands.

Subfinder (Discovery)
subfinder -d TARGET -all -recursive -silent -o subs.txt
FFUF (Directory Fuzzing)
ffuf -u https://TARGET/FUZZ -w /path/to/wordlist.txt -mc 200,301 -recursion

Workflow: Effective Recon

  • Run subfinder to identify all unique subdomains associated with the target.
  • Use httpx or httprobe to find live web servers among the subdomains.
  • Run ffuf against live subdomains to find hidden files, config directories, or admin panels.
  • Analyze the Burp Suite Target tab to build a site map and look for high-interest endpoints.

Insecure Direct Object Reference

Access unauthorized data by manipulating identifiers.

Dork: Find Profile URLs
site:TARGET inurl:'/profile' OR inurl:'/user' OR inurl:'/api/v1/'
Dork: Account Details
site:TARGET 'account' | 'settings' | 'billing' ext:php | ext:aspx | ext:jsp

Exploit Guide: Burp Suite IDOR

  • Log in with Account A and navigate to a private resource (e.g., View Profile).
  • Intercept the request in Burp Proxy and send to Repeater.
  • Log in with Account B in a different browser/private tab to find a valid ID (e.g., user_id=1234).
  • Go to Repeater, replace Account A's ID with Account B's ID.
  • If the response contains Account B's private data, IDOR is confirmed.

SQL Injection (SQLi)

Detect and exploit database vulnerabilities.

SQLMap: Basic Scan
sqlmap -u 'https://TARGET/product.php?id=10' --dbs --batch --random-agent
SQLMap: From Burp Request
sqlmap -r request.txt --level 3 --risk 2 --dbms=mysql --dbs

Exploit Guide: Burp to SQLMap

  • Locate a dynamic parameter in Burp Suite and send to Repeater.
  • Test manually by injecting a single quote '. Look for SQL syntax errors.
  • Right-click the request in Burp and select "Copy to file", save it as request.txt.
  • Pass this file to SQLMap using the -r flag to automate the data extraction.
  • Check if the application is vulnerable to Boolean-based, Error-based, or Time-based Blind SQLi.

Exploit Guide: Burp Collaborator XSS

  • Go to Burp -> Collaborator and click "Copy to clipboard" to get your unique URL.
  • Inject the Blind XSS payload into every input field: Contact forms, User Agents, Profile names.
  • Wait for a few hours/days. If a staff member views your payload, your Collaborator will ping.
  • Check the Collaborator tab for HTTP/DNS requests to capture the victim's cookies and IP.

SSRF & CSRF

Server-side request forgery and cross-site forgery.

Dork: SSRF Parameters
site:TARGET inurl:url= OR inurl:dest= OR inurl:redirect= OR inurl:uri=
SSRF Payload (AWS Metadata)
http://169.254.169.254/latest/meta-data/

Exploit Guide: CSRF PoC Generation

  • Find a state-changing action (e.g., "Change Email") that does not have a unique CSRF token.
  • Capture the request in Burp Suite Proxy.
  • Right-click the request -> Engagement tools -> Generate CSRF PoC.
  • Select Options -> Include auto-submit script and click "Test in browser".
  • Copy the URL and open it in a logged-in browser. If the email changes automatically, CSRF is valid.
